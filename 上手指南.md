# 上手指南（写给不想碰命令行的你）

这个文件夹已经是一个**接线完成、构建验证过**的 2CY 新基座。你只需要做两件事：把它放上 GitHub，然后等机器人（CI）帮你编译出可分发的程序。

## 第一步：放上 GitHub（两条路挑一条）

**路 A · GitHub Desktop（推荐，全程点鼠标）**
1. 下载安装 GitHub Desktop（desktop.github.com），登录你的 GitHub 账号；
2. 菜单 File → Add local repository → 选择这个文件夹；如果它提示「不是仓库，要不要创建」，点 create a repository，一路默认；
3. 左侧填一句提交说明（比如「2CY 新基座 v1 起步」），点 Commit；
4. 点顶部 Publish repository，去掉「Keep this code private」勾选与否随你，点 Publish。完成。

**路 B · 命令行（三行，复制粘贴）**
```bash
git add -A && git commit -m "2CY 新基座 v1 起步"
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main
```

## 第二步：发正式版（推一个版本号标签即可）

在仓库页面 Releases → Draft a new release → Choose a tag → 输入 `v1.0.0-alpha.2` 这样的版本号 → Publish。CI 会自动编译三平台并把带中文启动脚本的 zip 挂到该 Release 页面，用户从 Releases 页永久下载。以下为旧的 CI 激活说明留档：

CI 已激活（配置在 .github/workflows/2cy-build.yml）。以下留档备查：

**方法一（点鼠标）**：打开仓库页面 → 右上 Add file → Create new file → 文件名一字不差地输入 `.github/workflows/2cy-build.yml` → 把仓库里 `ci/2cy-build.yml` 的内容整个复制粘贴进去 → 点 Commit changes。

**方法二（给我一个新 token）**：重新生成一个 token，勾选 `repo` **和 `workflow`** 两个权限，发给我，我来放置并触发。

激活后，顶部 **Actions** 标签里会出现 `2cy-build` 任务（首跑约 10–20 分钟）。跑完变绿后点进去，页面底部 **Artifacts** 里就有 Windows / macOS / Linux 三个平台的程序包。

首跑变红也别慌：点进红色任务，把日志页面的链接发给我，我来修。

## 怎么让她带上人设（Phase 2 已上线）

把你的角色卡文件（旧产品导出的 `角色名.2cy.json` 就行）改名为 `character.2cy.json`，放到这个位置：

- Windows：`C:\Users\你的用户名\.local\share\2cy\character.2cy.json`
- macOS / Linux：`~/.local/share/2cy/character.2cy.json`

（`2cy` 文件夹不存在就手动建一个。）放好后**新开一个会话**即生效——她会以卡片上的人设说话，干活时照常使用全部工具。想换角色就换文件，想摘掉人设就删掉文件。注意：分身（后台子任务）不会带人设，这是刻意设计，省 token 也不污染检索结果。她记住的事存在同目录的 `memory.json`，随时可以打开查看或删除。

不想碰文件也行——这些都能在对话里直接说：「把你的口癖改成××」（她会更新自己的角色卡）、「你都记住我什么了」（列出画像和全部事实）、「忘掉第 3 条」「把对我的印象改成××」。每次动手前她都会走权限确认。
